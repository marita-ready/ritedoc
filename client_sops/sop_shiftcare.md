# Standard Operating Procedure: ShiftCare Integration with RiteDoc

This Standard Operating Procedure (SOP) details the click-by-click process for exporting progress notes from ShiftCare, processing them through the RiteDoc compliance engine, and re-importing the audit-prepared drafts back into ShiftCare.

## 1. Exporting Progress Notes from ShiftCare

The first step is to extract the raw support worker progress notes from ShiftCare in a format compatible with RiteDoc.

### 1.1 Accessing the Events Report

1.  Log in to your ShiftCare administrator account.
2.  Navigate to the main menu on the left-hand side of the screen.
3.  Scroll down and click on the **Reports** tab.
4.  In the drop-down menu that appears, scroll to the bottom and select the **Events** tab.
5.  From the subsequent drop-down, select **Details** to access the detailed Events Report [1].

### 1.2 Filtering the Data

1.  On the Details Events Report page, locate the filter options at the top of the screen.
2.  Use the **Date Filtering** option to select the specific time frame for the progress notes you wish to process (e.g., "Last Week" or a custom date range).
3.  Ensure the **Client/Staff Filtering** is set appropriately to capture the relevant notes.
4.  Under the category filters, ensure the blue tick is selected next to the categories representing progress notes or case notes.
5.  Click the **Apply** button to activate the filters and update the report view [1].

### 1.3 Exporting to CSV

1.  Once the report is filtered correctly, navigate to the bottom right corner of the page.
2.  Locate the two icons: a download (export) icon and a print icon.
3.  Click the **Download (export)** icon.
4.  The report will be exported and downloaded to your local machine as an Excel/CSV file [1].
5.  Locate the downloaded file in your computer's "Downloads" folder.

## 2. Processing Notes in RiteDoc

With the raw data exported, you will now use RiteDoc to scrub PII and generate audit-prepared drafts.

### 2.1 Importing into RiteDoc

1.  Open the RiteDoc desktop application on your local machine.
2.  On the main dashboard, click the **Import CSV** button.
3.  In the file browser window that appears, navigate to your "Downloads" folder and select the CSV file exported from ShiftCare.
4.  Click **Open** to load the file into RiteDoc.

### 2.2 Running the Compliance Engine

1.  Once the file is loaded, RiteDoc will display a summary of the imported notes.
2.  Click the **Process Notes** button to initiate the compliance engine.
3.  RiteDoc will automatically perform the following steps offline:
    *   **PII Scrubbing:** The Nanoclaw scrubber will identify and redact all personally identifiable information, replacing it with standardised tags (e.g., `[Participant]`, `[Support Worker]`).
    *   **AI Rewrite:** The local AI model will rewrite the raw notes into professional, audit-prepared drafts based on the active state cartridge.
    *   **Compliance Scoring:** The engine will evaluate each note against the 5-pillar rubric and assign an internal score.
    *   **Traffic Light Assignment:** Based on the score and identified red flags, each note will be assigned a traffic light indicator (GREEN, ORANGE, RED).

### 2.3 Reviewing the Output

1.  After processing is complete, RiteDoc will display the results in a clear, tabular format.
2.  Review the traffic light indicators:
    *   **GREEN:** The note is audit-ready and requires no further action.
    *   **ORANGE:** The note requires additional information or clarification. Review the bracketed flags (e.g., `[UPPERCASE FIELD — plain English explanation]`) and update the note accordingly.
    *   **RED:** A critical incident or severe compliance risk has been detected. These notes will float to the top of the list for immediate attention.
3.  Make any necessary manual edits directly within the RiteDoc interface.

### 2.4 Exporting the Processed Drafts

1.  Once you are satisfied with the processed notes, click the **Export Processed CSV** button.
2.  Choose a secure location on your local machine to save the new CSV file containing the audit-prepared drafts.

## 3. Re-importing into ShiftCare

The final step is to update the original records in ShiftCare with the processed, compliant notes.

### 3.1 Using the CSV Import Tool

1.  Return to your ShiftCare administrator account.
2.  Navigate to the **Integrations Marketplace** or the relevant data management section.
3.  Locate and select the **CSV Import** tool [2].
4.  Follow the on-screen instructions to upload the processed CSV file generated by RiteDoc.
5.  Map the columns in the CSV file to the corresponding fields in ShiftCare (e.g., mapping the "Processed Note" column to the "Progress Note" field).
6.  Initiate the import process to update the records in bulk.

*Note: The exact menu path for the CSV Import tool in ShiftCare may vary depending on your specific account configuration and the version of the software. Please consult the ShiftCare Help Center or your account manager if you cannot locate the tool.*
