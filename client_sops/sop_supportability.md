# Standard Operating Procedure: SupportAbility Integration with RiteDoc

This Standard Operating Procedure (SOP) details the click-by-click process for exporting progress notes from SupportAbility, processing them through the RiteDoc compliance engine, and re-importing the audit-prepared drafts back into SupportAbility.

## 1. Exporting Progress Notes from SupportAbility

The first step is to extract the raw support worker progress notes from SupportAbility in a format compatible with RiteDoc. SupportAbility provides an Activity Report for this purpose.

### 1.1 Accessing the Activity Report

1.  Log in to your SupportAbility administrator account.
2.  Navigate to the main Dashboard.
3.  Click on the **Reports** menu [1].
4.  Select **Activity Report** from the available options [1].

### 1.2 Filtering the Data

1.  On the Activity Report page, locate the search criteria options.
2.  Use the **Activity Date** filter to select the specific time frame for the progress notes you wish to process (e.g., "From" and "To" dates) [1].
3.  Use the **Staff Members and Clients** filters to select specific individuals if necessary [1].
4.  Use the **Sites and Services** filters to narrow down the results by location or service type [1].
5.  Use the **Activity Sign Off Status** filter to select activities that have been completed but not yet signed off, or those that require review [1].
6.  Click the **Generate Report** button to view the list results [1].

### 1.3 Exporting to CSV

1.  Once the report is generated and displaying the correct data, locate the **Actions** menu on the top right-hand side of the window [1].
2.  Click the **Actions** menu.
3.  Select the option to **Download CSV Export** [1].
4.  The file will be generated and downloaded to your local machine. Locate the downloaded file in your computer's "Downloads" folder.

## 2. Processing Notes in RiteDoc

With the raw data exported, you will now use RiteDoc to scrub PII and generate audit-prepared drafts.

### 2.1 Importing into RiteDoc

1.  Open the RiteDoc desktop application on your local machine.
2.  On the main dashboard, click the **Import CSV** button.
3.  In the file browser window that appears, navigate to your "Downloads" folder and select the CSV file exported from SupportAbility.
4.  Click **Open** to load the file into RiteDoc.

### 2.2 Running the Compliance Engine

1.  Once the file is loaded, RiteDoc will display a summary of the imported notes.
2.  Click the **Process Notes** button to initiate the compliance engine.
3.  RiteDoc will automatically perform the following steps offline:
    *   **PII Scrubbing:** The Nanoclaw scrubber will identify and redact all personally identifiable information, replacing it with standardised tags (e.g., `[Participant]`, `[Support Worker]`).
    *   **AI Rewrite:** The local AI model will rewrite the raw notes into professional, audit-prepared drafts based on the active state cartridge.
    *   **Compliance Scoring:** The engine will evaluate each note against the 5-pillar rubric and assign an internal score.
    *   **Traffic Light Assignment:** Based on the score and identified red flags, each note will be assigned a traffic light indicator (GREEN, ORANGE, RED).

### 2.3 Reviewing the Output

1.  After processing is complete, RiteDoc will display the results in a clear, tabular format.
2.  Review the traffic light indicators:
    *   **GREEN:** The note is audit-ready and requires no further action.
    *   **ORANGE:** The note requires additional information or clarification. Review the bracketed flags (e.g., `[UPPERCASE FIELD — plain English explanation]`) and update the note accordingly.
    *   **RED:** A critical incident or severe compliance risk has been detected. These notes will float to the top of the list for immediate attention.
3.  Make any necessary manual edits directly within the RiteDoc interface.

### 2.4 Exporting the Processed Drafts

1.  Once you are satisfied with the processed notes, click the **Export Processed CSV** button.
2.  Choose a secure location on your local machine to save the new CSV file containing the audit-prepared drafts.

## 3. Re-importing into SupportAbility

The final step is to update the original records in SupportAbility with the processed, compliant notes.

### 3.1 Using the Import Function

1.  Return to your SupportAbility administrator account.
2.  Navigate to the relevant data import section or settings menu.
3.  Locate the option to import or update activity records or client support logs.
4.  Follow the on-screen instructions to upload the processed CSV file generated by RiteDoc.
5.  Map the columns in the CSV file to the corresponding fields in SupportAbility (e.g., mapping the "Processed Note" column to the "Activity Notes" field).
6.  Ensure you use a unique identifier (like the Activity ID) to match the records accurately and update existing entries rather than creating duplicates.
7.  Initiate the import process to update the records in bulk.

*Note: The exact menu path for the import function in SupportAbility may vary depending on your specific account configuration and the version of the software. Please consult the SupportAbility Knowledge Base or your account manager if you cannot locate the tool.*
