Upload New Cartridge Version

Upload a pre-signed .rsp (Regulation Sync Package) file produced by tools/sign_cartridge.js on Aroha (Marita's laptop).

The private signing key lives only on Aroha. Even if this dashboard is compromised, no fake updates can be pushed.

How to create a .rsp package on Aroha:

  1. Place your cartridge JSON files in a directory (e.g. ~/cartridge-update/)
  2. Run: node tools/sign_cartridge.js ~/cartridge-update/ 2.1.0 "Release notes"
  3. This creates cartridge_v2.1.0.rsp — upload that file here

API Configuration

The ReadyCompliant API is a Cloudflare Worker. All dashboard operations and RiteDoc regulation sync go through this single endpoint.

Worker API URL https://readycompliant-api.workers.dev
Auth Header Authorization: Bearer <jwt_token>
Regulation Sync POST /api/sync/check — HMAC-SHA256 authenticated

Available API Endpoints

POST
/api/auth/login
Admin dashboard login (returns JWT)
curl -X POST 'https://api.readycompliant.com/api/auth/login' \
  -H 'Content-Type: application/json' \
  -d '{"email":"admin@readycompliant.com","password":"..."}'
GET
/api/clients
List all clients
curl 'https://api.readycompliant.com/api/clients' \
  -H 'Authorization: Bearer <token>'
POST
/api/keys/generate
Generate activation keys
curl -X POST 'https://api.readycompliant.com/api/keys/generate' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"subscription_type":"standard","count":1}'
POST
/api/sync/check
Regulation sync check (called by RiteDoc app — HMAC authenticated)
curl -X POST 'https://api.readycompliant.com/api/sync/check' \
  -H 'Content-Type: application/json' \
  -H 'X-HMAC-Signature: <hmac-sha256-hex>' \
  -H 'X-Timestamp: <unix-timestamp>' \
  -d '{"license_key":"RD-XXXX-XXXX-XXXX","hardware_fingerprint":"...","current_version":"2.0.0"}'
POST
/api/cartridges/upload
Upload a pre-signed .rsp cartridge package (Ed25519 signature verified)
curl -X POST 'https://api.readycompliant.com/api/cartridges/upload' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d @cartridge_v2.1.0.rsp
GET
/api/stats/overview
Dashboard overview stats (MRR, active clients, etc.)
curl 'https://api.readycompliant.com/api/stats/overview' \
  -H 'Authorization: Bearer <token>'
POST
/api/mobile/verify-code
Verify and redeem a mobile access code (public — called by mobile app)
curl -X POST 'https://api.readycompliant.com/api/mobile/verify-code' \
  -H 'Content-Type: application/json' \
  -d '{"code":"MAC-XXXX-XXXX-XXXX","device_id":"optional-device-id"}'
POST
/api/mobile/codes/generate
Generate mobile access codes for a BIAB agency
curl -X POST 'https://api.readycompliant.com/api/mobile/codes/generate' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"agency_id":1,"count":5}'
GET
/api/mobile/codes
List mobile access codes (optional ?agency_id= filter)
curl 'https://api.readycompliant.com/api/mobile/codes?agency_id=1' \
  -H 'Authorization: Bearer <token>'
POST
/api/seat-requests
Submit a wholesale seat request for an agency
curl -X POST 'https://api.readycompliant.com/api/seat-requests' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"agency_id":1,"seats_requested":10}'
PUT
/api/seat-requests/:id/approve
Approve a pending seat request (generates activation keys)
curl -X PUT 'https://api.readycompliant.com/api/seat-requests/1/approve' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"admin_notes":"Approved for Q2 rollout"}'
PUT
/api/seat-requests/:id/reject
Reject a pending seat request
curl -X PUT 'https://api.readycompliant.com/api/seat-requests/1/reject' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"admin_notes":"Exceeds allocation limit"}'
POST
/api/webhooks/stripe
Stripe webhook handler (auto-generates keys on payment)
# Configure in Stripe Dashboard:
# Webhook URL: https://api.readycompliant.com/api/webhooks/stripe
# Events: payment_intent.succeeded, customer.subscription.deleted
GET
/api/client-assignments
List client assignments (optional ?agency_id= and ?status= filters)
curl 'https://api.readycompliant.com/api/client-assignments?agency_id=1' \
  -H 'Authorization: Bearer <token>'
POST
/api/client-assignments
Assign an activation key to a client
curl -X POST 'https://api.readycompliant.com/api/client-assignments' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"agency_id":1,"client_name":"Jane Doe","client_email":"jane@example.com","activation_key":"RD-XXXX-XXXX-XXXX-XXXX"}'
PUT
/api/client-assignments/:id/revoke
Revoke a client assignment (optionally deactivate the key)
curl -X PUT 'https://api.readycompliant.com/api/client-assignments/1/revoke' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"reason":"Client no longer active","deactivate_key":false}'
POST

GET

POST
